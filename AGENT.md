## 專案概念

專案名稱：呷胖環遊日本。

這是一款手機直式優先、離線可玩的 JLPT 單字拼字遊戲。玩家透過 4×4 假名棋盤點選假名拼出 N5 單字，幫圓滾滾水彩貪吃柴「呷胖」通過沖繩站點並收集明信片。

Phase 1 只做 N5 沖繩區，重點是核心可玩迴圈穩定。

## 技術邊界

- 起步使用單檔 `index.html`。
- 不引入後端。
- 不做帳號系統。
- 不依賴網路才能遊玩。
- 不載入 Google Fonts、CDN 或其他外部執行期資源；離線冷啟動也不能卡外部請求。
- 圖片資源放在 `assets/`。
- 明信片圖片缺檔時要顯示 fallback。
- 所有依賴如果需要安裝，必須安裝在專案資料夾內，不要裝到系統全域。

## Phase 1 核心迴圈

```text
進入沖繩站點
→ 產生 4×4 平假名棋盤
→ 玩家點選棋盤假名拼字
→ 命中 N5 單字
→ 顯示單字卡 5 秒
→ 累積站點進度
→ 拼滿 10 個 N5 單字
→ 過站演出
→ 取得明信片
→ 進下一站
```

## 假名規則

- 棋盤只放平假名。
- 棋格必須支援滑鼠、觸控與鍵盤 Enter/Space 選字。
- 片假名詞以平假名讀音正規化後比對。
- 小字 `ゃ`、`ゅ`、`ょ`、`っ` 是獨立棋格。
- 濁音與半濁音可以出現在棋盤。
- 長音符 `ー` 不直接進棋盤，會由 kana normalizer 轉為前一個假名對應的母音。
- 站點食物顯示文字可以包含片假名與 `ー`，不要誤刪。

範例：

```text
ばす → バス
てれび → テレビ
ぱん → パン
がっこう → 学校
```

## 字典資料注意事項

每筆單字至少包含：

```js
{
  id: "n5_0001",
  reading: "バス",
  normalizedReading: "ばす",
  writing: "バス",
  zh: "公車",
  level: "N5",
  script: "katakana"
}
```

重要規則：

- 同音詞全收，不要合併。
- 已拼過清單記錄 `id`，不要只記 `normalizedReading`。
- `normalizedReading` 是遊戲命中的主要 key，但必須由 `normalizeKanaReading(reading)` 產生，不要只相信人工填入欄位。
- `reading` 與 `writing` 是顯示用資料，不要拿來直接做棋盤比對。

## 存檔

使用 `localStorage`。

建議 key：

```js
const SAVE_KEY = "fatShiba.phase1.save.v1";
```

需要保存：

- 目前站點。
- 已完成站點。
- 已取得明信片。
- 已拼過單字 id。
- 各站目前已累積的單字數。

存檔損壞時，應回到初始狀態，不要讓畫面白掉。

## UI 與體驗要求

- 手機直式優先。
- 按鈕、棋格要適合手指點擊。
- 4×4 棋盤尺寸穩定，不要因選取狀態或提示狀態跳動。
- 玩家進畫面後應能直接開始玩，不要先看到大段說明。
- 不要用純文字塞滿畫面；必要提示短而清楚。
- 美術方向是溫暖水彩、留白多，但操作要明確。

## 開發與驗證習慣

修改前：

- 先看相關檔案。
- 先理解現有資料結構與流程。
- 不要為了新功能重寫不相關區塊。

修改後：

- 至少做語法檢查或 smoke test。
- 若有 dev server，測試完要停止程序。
- 若需保留 server 給使用者操作，必須告知網址與關閉方式。

建議驗收重點：

```text
□ がっこう → 学校
□ ちょっと 可命中
□ ばす → バス
□ 每局 3 到 5 個保底詞
□ 每站 10 個 N5 單字過站
□ 10 站可通關
□ 明信片缺圖 fallback 正常
□ localStorage 重新整理後仍保存
```

## 不要在 Phase 1 做的事

- 混合平假名／片假名棋盤。
- 長音 `ー` 的完整拼字教學。
- N4 以上詞庫。
- 後端 API。
- 登入帳號。
- 排行榜。
- 多人功能。
- 雲端同步。
- 大型框架遷移。

先把核心玩法做穩，再擴張。這個專案最容易失控的地方是「看起來每個功能都很可愛」，但可愛功能太多的時候，bug 也會一起可愛地繁殖。
